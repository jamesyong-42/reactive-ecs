API reference

Every exported function, World member, scheduler, and type. For concepts and patterns, see the guide. All subscription methods return an Unsubscribe function; all observers fire synchronously inside the mutating call.

Definition#

defineComponent#

Defines a component type. defaults sets both the TypeScript shape and the initial values merged under data passed to addComponent. Data can be any shape — numbers, strings, arrays, nested objects, class instances. Names must be unique across all defineComponent calls in a world; a collision throws on first store access. Returns a frozen identity object.

defineTag#

Defines a zero-data boolean marker — presence is the information (Selected, Hovered, Dirty).

defineRelation#

Defines a relation — a managed, inverse-indexed, lifecycle-cleaned edge between two entities. See RelationOptions for exclusivity bounds and the onTargetDestroy policy. The returned object carries the normalized options (defaults applied).

defineResource#

Defines a world-level singleton (camera, config, a spatial-index instance). The store is created lazily from defaults on first access.

defineSystem#

Defines a named system. after / before (string or string[]) constrain ordering within a scheduler; phase assigns the system to a PhasedScheduler phase and is ignored by SystemScheduler. See SystemDef.

Not#

Wraps a component or tag as a negated query term: query(A, Not(B)) matches entities that have A and do not have B. A type whose store was never created counts as absent. Negated terms participate in incremental cache maintenance like positive ones.

createWorld#

Creates an independent world — entities, component/tag/relation stores, resources, query caches, and subscriptions are all per-world.

tickWorld#

The blessed frame protocol in one call: runs fn(world) (your systems — typically (w) => scheduler.execute(w)), then emitFrame(), clearDirty(), incrementTick() in the recommended order. Equivalent to the four manual calls; harder to get wrong.

World · properties#

currentTick#

Current tick number; advanced by incrementTick.

entityCount#

Number of live entities.

mutationOrigin#

Origin tag of the current mutation window — undefined outside any withOrigin window (the implicit "local" origin). Because every observer fires synchronously inside the mutating call, a handler reading this always sees the origin of exactly the mutation that fired it. Readable anywhere, not just in handlers.

World · entities#

createEntity#

Creates an entity and returns its id. Ids are allocated from a monotonic counter and never reused. Fires onEntityCreated. O(1).

createEntityWithId#

Creates an entity under a caller-chosen id — the restore primitive for id-preserving deserialization. On success the counter advances to id + 1 and onEntityCreated fires; observably identical to a createEntity() that landed on id.

  • Throws if id is not a positive integer, is already alive, or is below the internal counter — ids are never reused, so restore must create entities in ascending id order.
  • Restores into a fresh world only; it cannot resurrect a destroyed id in a live one. For undoable deletion prefer a tombstone tag (see the guide).

setNextEntityId#

Restores the id counter exactly, so subsequent createEntity() calls allocate from n. The counter only moves forward — throws if n is not an integer or is below the current counter. Call after the restore loop: the saved counter can exceed the highest restored id (entities destroyed before the save consumed ids), and restoring it keeps stale references stale forever.

destroyEntity#

Destroys an entity, removing all its components, tags, and relation edges (as source or target). No-op if already dead. The observable sequence: onEntityDestroyed listeners (data still readable) → query caches drop the id → relation sweep (onRelationRemoved per edge, policy effects collected) → per-component onComponentRemoved + buffers → per-tag onTagRemoved + buffers → deferred onTargetDestroy effects apply (cascades, tags). See the guide for the full contract.

entityExists#

true while alive — and false forever after destroy. A stale id can never alias a new entity; no generational indices needed.

World · components#

addComponent#

Attach-or-replace. data is optional and partial — shallow-merged over the type's defaults; plain objects and arrays are deep-cloned at any depth (class instances kept by reference), so two entities never share nested state. On first attach, observers fire with prev === undefined and the id lands in queryAdded + queryChanged. On an entity that already has the component, the value is replaced and observers fire with prev = the old value, landing in queryChanged only — so prev === undefined reliably means "first attach". Throws if the entity is not alive.

removeComponent#

Detaches a component. Fires onComponentRemoved before the data is deleted (so prev is readable) and lands in the queryRemoved buffer. No-op if absent.

getComponent#

Current value, typed read-only, or undefined if not present. The returned object is the live store value — mutating it in place is invisible to the world; write through setComponent or replaceComponent.

hasComponent#

Presence check.

setComponent#

Shallow-merges a partial update over the current value — one level deep: nested objects in data replace, they don't merge into. Incoming plain data is defensively cloned (caller-held aliases can't mutate world state later). Fires onComponentChanged with the previous value and lands in the queryChanged buffer. Silent no-op if the entity doesn't have the component (or is dead) — use addComponent to attach.

replaceComponent#

Full-value upsert: stores a defensive clone of data (over defaults), replacing whatever was there. Emits the true prev (undefined if absent); buffers: queryAdded + queryChanged when attaching, queryChanged only when replacing. Throws if the entity is not alive. The unambiguous alternative to setComponent's shallow merge when nested data is involved.

World · tags#

addTag#

Adds a tag. Fires onTagAdded; lands in the queryAddedTag buffer. No-op if already present.

removeTag#

Removes a tag. Fires onTagRemoved; lands in the queryRemovedTag buffer. No-op if absent.

hasTag#

Presence check.

World · relations#

relate#

Adds an edge from source to target.

  • Throws if source or target is not alive — a born-dangling edge is impossible.
  • No-op if the exact edge already exists.
  • Exclusivity replaces: if a bound would be violated, the existing edge is removed first — removed-then-added events fire, mirroring setComponent overwrite semantics.
  • Lands in the queryRelationAdded buffer (net-cancelling) and fires onRelationAdded.

unrelate#

Removes the edge to target — or, with target omitted, all of source's outgoing edges for this relation. No-op if absent. Lands in queryRelationRemoved (net-cancelling) and fires onRelationRemoved per edge.

getTargets#

The targets source points at. [] if none.

getTarget#

Source's single target — convenience for sourceExclusive relations.

getSources#

The sources pointing at target — the always-coherent inverse index, maintained by the store, never by you. [] if none.

World · resources#

getResource#

Reads a singleton. The store is created lazily from the type's defaults on first access.

setResource#

Shallow-merges a partial update over the singleton. Incoming plain data is defensively cloned, same as setComponent.

World · queries#

query() results are cached per type-combination and maintained incrementally as components, tags, and entities change — repeat calls do no scanning. The per-tick buffers (queryChanged / queryAdded / queryRemoved and their tag/relation twins) accumulate until clearDirty and are net-cancelling: an add and remove of the same thing within one tick appears in neither buffer.

query#

Entities matching all positive components/tags and none of the Not() terms. With no arguments, returns all live entities. Throws if all terms are negated — a pure-negative query would scan every entity; keep at least one positive term.

queryTagged#

All entities with the tag.

queryChanged#

Entities whose component was written this tick — adds included: a newly added component appears in both queryAdded and queryChanged, so consumers handling both must dedupe.

queryAdded#

Entities that received the component this tick. Net-cancels with removeComponent in the same tick.

queryRemoved#

Entities that lost the component this tick — via removeComponent or destroyEntity. The single channel for "this entity no longer has C"; consumers managing external resources need no separate destroy bookkeeping. The buffer holds ids only — to read the dying value, use onComponentRemoved.

queryAddedTag#

Entities that gained the tag this tick. Net-cancels with removeTag.

queryRemovedTag#

Entities that lost the tag this tick — via removeTag or destroyEntity. Net-cancels with addTag.

queryRelation#

All live edges as [source, target] pairs.

queryRelationAdded#

Edges added this tick. Net-cancels with unrelate of the same edge.

queryRelationRemoved#

Edges removed this tick — via unrelate, exclusivity replacement, or destroyEntity of either endpoint. Net-cancels with relate of the same edge.

queryChangedResources#

Every resource type written via setResource this tick, in first-changed order, deduped. Lazy store creation does not count. Cleared by clearDirty().

World · events#

All handlers fire synchronously inside the mutating call, in subscription order; per-entity (or per-source) handlers fire before wildcard handlers. Every method returns an Unsubscribe function.

onComponentChanged#

Fires on every write — including the initial addComponent, where prev === undefined, so one subscription covers "appeared" and "updated". Pass entityId to observe a single entity — the shape a useComponent() hook needs.

onComponentRemoved#

Fires before the data is deleted — prev is the value being torn down, the right moment to release GPU buffers, DOM nodes, or subscriptions keyed by it. Also fires for each component an entity owned at the moment destroyEntity was called.

onTagAdded#

Fires when the tag is added; optional per-entity filter.

onTagRemoved#

Fires when the tag is removed — including during destroyEntity.

onRelationAdded#

Fires when an edge is added. A bare id filters by source (matching the entityId filter on component events); { target } filters by target — the parent-watches-children shape; { source, target } matches the exact edge. Emit order: per-source → per-target → wildcard.

onRelationRemoved#

Fires when an edge is removed — via unrelate, exclusivity replacement, or destroyEntity of either endpoint. During a destroy the dying entity's components and tags are still readable from the handler, but handlers must not mutate the world mid-destroy: read freely, react from a system over queryRelationRemoved instead.

onResourceChanged#

Fires synchronously inside every setResource, after the merge — prev is a top-level pre-merge snapshot (the object itself never aliases next; nested values the merge didn't replace are shared — safe, since stored values are only ever replaced, never mutated in place), next is the live post-merge value. No per-entity filter (resources are singletons). Subscribing lazily creates the store. Handlers can read mutationOrigin.

onEntityCreated#

Fires on createEntity and createEntityWithId.

onEntityDestroyed#

Fires first in the destroy sequence — the entity's components, tags, and relations are still fully readable, so callbacks can snapshot whatever they need.

onFrame#

Fires on every emitFrame — the end-of-tick hook for flushing batched UI updates.

World · origin#

withOrigin#

Tags every mutation made synchronously inside fn with origin, readable by handlers via mutationOrigin — the echo-suppression primitive for modules that both observe and mutate (sync adapters, undo journals). The library attaches no semantics to any origin; the vocabulary is yours (symbols recommended).

  • Throws unless origin is a string or symbol — undefined stays unforgeable as "no origin".
  • Re-entrant: nested calls stack; innermost wins; exiting restores the enclosing origin — also on throw.
  • Returns fn's return value.
  • Synchronous only: if fn is async, mutations after the first await are not tagged — re-enter withOrigin in continuations.
  • destroyEntity cascades — teardown events and relation policy effects — inherit the origin active at the call site. Per-tick buffers stay origin-blind.

World · introspection#

All O(k) in the number of registered types or attached items — no hidden scan of entity storage.

getAllEntities#

All live entity ids.

getRegisteredComponents#

Every ComponentType that has had a store created in this world.

getRegisteredTags#

Every TagType that has had a store created in this world.

getRegisteredResources#

Every ResourceType that has been accessed in this world.

getComponentsOf#

The ComponentTypes currently attached to an entity — combine with getComponent to walk all data generically (serializers, inspectors).

getTagsOf#

The TagTypes currently attached to an entity.

World · frame lifecycle#

The end-of-frame protocol, called in this order after your systems run:

emitFrame#

Fires every onFrame subscriber.

clearDirty#

Resets all per-tick buffers — component changed / added / removed, tag added / removed, and relation added / removed. Buffers accumulate until you call this; skipping frames simply batches more change.

incrementTick#

Advances currentTick by one.

SystemScheduler#

Orders systems by after / before constraints using a stable topological sort (Kahn's algorithm; registration order breaks ties). Cycles throw with the names involved; constraints referencing unregistered names are tolerated and ignored.

register#

Adds a system. Re-registering a name replaces the existing system (hot-reload friendly) and invalidates the cached sort.

remove#

Removes a system by name.

execute#

Runs all systems in dependency order, bracketing each with the profiler's beginSystem / endSystem if attached.

getSystemNames#

The resolved execution order (triggers the sort).

size#

Number of registered systems.

profiler#

Optional instrumentation hook — see SystemProfiler.

PhasedScheduler#

Runs systems in a caller-defined phase order — the library ships zero phase opinions. Within a phase, after / before topo-sort identically to SystemScheduler (one runs internally per phase). Across phases, ordering is implicit in phase order; cross-phase constraints that contradict phase order are rejected at first execute() with an error explaining the fix (redundant ones — an after pointing at an earlier phase — are tolerated, since phase order already satisfies them). The constructor throws on an empty or duplicate-containing phases array, or a defaultPhase not in it. Use as const on the phases literal for type-narrowed phase strings throughout. Also exposes remove(name), getSystemNames(), and profiler with the same shapes as SystemScheduler.

register#

Adds a system to the bucket for system.phase, falling back to defaultPhase. Throws if the system has no phase and no default is configured, or if the phase is not in the configured list. Re-registering into a different phase moves the system — it never runs twice.

getPhase#

The phase a system is registered in, or undefined.

getPhases#

The configured phase order (read-only view of the constructor input).

execute#

Runs one tick, phase by phase. Non-empty phases are bracketed with the profiler's optional beginPhase / endPhase; systems with beginSystem / endSystem. Cross-phase constraint validation happens lazily on first call and after every register/remove.

Devtools#

Two subpath exports; the core . entry is untouched. @jamesyong42/reactive-ecs/devtools is headless and dependency-free; @jamesyong42/reactive-ecs/devtools/react needs React (optional peer, ≥18). See the guide for the quick start.

createLifecycleRecorder#

Captures every entity's birth and death from the event hooks (not a poll), so entities born and destroyed within one tick are recorded. Attach before entities spawn. At death the entity's descriptor is frozen while its data is still readable (destroyEntity fires listeners before teardown). cap (default 4000) bounds memory by evicting the oldest dead records — never a living entity.

defaultDescriber#

Zero-config identity off composition: label from the first component's name (else first tag, else 'entity'), remaining tags as detail, default color, no outcome. The fallback when no custom describer is passed.

formatValue / formatComponentData#

The inspector's compact value formatting (x=10 y=20), exported for custom panels.

EcsInspector#

Draggable, resizable, layout-persisting floating window: live entity list (tags + component values at ~16 Hz), plus the timeline tab when a recorder is provided. Styles are injected once into a deduped <style> tag — no CSS import.

EntityTimeline#

Canvas-rendered waterfall of entity lifecycles — one bar per entity, birth to death (or the live "now" edge). Pan (drag / two-finger scroll), pinch-zoom the time axis, live-tail that re-engages at the right edge, wall-clock or tick units, hover tooltip. Bar colors come from the describer; outcome: 'win' | 'lose' draws green/red end-caps. Usable standalone or inside the inspector.

EntityDescriptor / EntityDescriber#

The one seam between the generic tools and your domain: identity read off composition. Pass a describer to get domain labels, colors, and outcome end-caps everywhere at once.

Types#

EntityId#

Opaque entity identifier — a sequential integer internally, monotonic, never reused.

ComponentType#

Frozen identity object from defineComponent; the name is the lookup key.

TagType#

Frozen identity object from defineTag.

ResourceType#

Frozen identity object from defineResource.

RelationType#

Frozen identity object from defineRelation, carrying the normalized options.

RelationOptions#

The two exclusivity bounds are independent: ChildOf is sourceExclusive only (one parent, many children); a true 1:1 ownership edge sets both. onTargetDestroy runs per source when the target dies, after the destroy sweep completes: 'clear' drops the edge, 'cascade' destroys the source too, { tag } adds the tag to the source.

NotTerm#

Negated query term from Not().

SystemDef#

runIf is evaluated immediately before the system would run, so it sees same-tick writes from earlier systems via the per-tick buffers. On a skip the profiler's optional skipSystem fires instead of beginSystem/endSystem. Caveat: buffers clear at end of tick — order lazy readers after their writers.

SystemProfiler#

Attach to either scheduler's profiler property; the scheduler just calls the hooks if present.
